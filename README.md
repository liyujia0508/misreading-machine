# Emotion Recognition System
## Affective Computing Homework

An end-to-end **Text Emotion Recognition** web application built on classical NLP and the circumplex model of affect.

---

### Demo Preview

| Feature | Details |
|---|---|
| **Model** | TF-IDF (1-3 grams, 100k features) + Logistic Regression |
| **Dataset** | [dair-ai/emotion](https://huggingface.co/datasets/dair-ai/emotion) — 20k English tweets |
| **Emotions** | sadness · joy · love · anger · fear · surprise |
| **Test Accuracy** | ~93 % |
| **Affect Theory** | Russell's (1980) Valence–Arousal Circumplex Model |

---

### Quick Start

**1. Install dependencies**
```bash
pip install -r requirements.txt
```

**2. One-click run** *(trains model automatically if needed)*
```bash
python run.py
```

**Or step by step:**
```bash
python train.py      # ~30s on modern hardware
python run.py        # opens http://localhost:5000
```

### Theoretical Background

**Emotion Categories** (Ekman-inspired, 6 basic emotions):
- **Sadness** — low valence, low arousal
- **Joy** — high valence, high arousal
- **Love** — high valence, medium arousal
- **Anger** — low valence, high arousal
- **Fear** — low valence, high arousal
- **Surprise** — medium valence, very high arousal

**Valence–Arousal Model** (Russell, 1980):
> Emotions are represented in a 2D continuous space: *valence* (pleasant ↔ unpleasant) and *arousal* (activated ↔ deactivated). The circumplex visualization maps each detected emotion onto this space.

### API Reference

**POST `/api/predict`**
```json
// Request
{ "text": "I am so happy today!" }

// Response
{
  "predicted_emotion": "joy",
  "confidence": 0.9271,
  "emoji": "😊",
  "color": "#FBBF24",
  "valence": 0.8,
  "arousal": 0.6,
  "probabilities": { "joy": {"probability": 0.9271, ...}, ... },
  "top_words": [{"word": "happy", "score": 0.412}, ...]
}
```

**GET `/api/health`** — Returns `{"status": "ok", "model_loaded": true}`
